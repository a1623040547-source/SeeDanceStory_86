## 二、素材类型与衔接规则（API 限制：仅 1 张图）

| 类型 | 说明 | 何时使用 |
|------|------|----------|
| **首图（段 1）** | 夏日海边、沙滩、两人度假服、86 画风；需先上传到图床得到 **公网 URL** | 段 1：image_urls 传此 URL + 段 1 提示词 |
| **上一段尾帧图** | 从上一段生成视频**最后一帧**截取；须上传到图床得到 URL | 段 2～8：image_urls 传该尾帧 URL + 本段提示词 |
| **角色形象** | 复用 section03/refs/characters；若需与首图一致，可将角色图与海边场景合成一张首图 | 首图可含两人定妆，或单独作绘制参考 |

**API 约束**：Seedance 2.0 API 的 `image_urls` 仅支持 **最多 1 张**图片；因此每段只能传「首图」或「上一段尾帧图」其一，不能同时传角色图+尾帧。建议首图即包含「海边+两人 86 画风」，后续段仅用尾帧图作视觉衔接。

### ⚠️ Seedance 传入图片的组成与用途（不能直接用作首帧）

当使用 **多图合成**（如 `scripts/composite_refs.py` 将 3 张人物 + 1 张环境拼成 2×2）作为段 1 的参考图时：

- **组成**：合成图是 2×2 参考拼图（[人物, 人物] / [人物, 环境]），并非「单一场景的首帧画面」。
- **用途**：仅作**参考图**——供 Seedance 理解画风、角色外形与场景氛围；模型根据**提示词**生成新的画面作为视频内容，而不是把这张拼图当第一帧播放。
- **不能直接用作首帧**：该图不能当作视频的第一帧，也不能作为段 2 的 V2V 输入。**段 2 及之后必须使用「段 1 生成视频的最后一帧」截取的尾帧图**（如 `section04_tail_01.png`）上传得 URL 后传入 API，才能保证画面衔接。

### 每段使用的素材一览与工作流程

**段 1**：一开始没有现成视频，故**必须先生图首帧**，再用首帧生视频；**本段视频生成后**，从该视频最后一帧截取得到尾帧（section04_tail_01.png）。段 1 尾帧不是「一开始」就有的。

| 段 | 生成方式 | 本段使用的素材 | 本段生成后需产出 |
|:--:|----------|----------------|------------------|
| 段 1 | 生图首帧 → 图+文生视频 | **首帧图**（生图得，见 phase01_img_first.txt） | 视频 + **截取**尾帧 → `section04_tail_01.png`，上传得 URL 供段 2 |
| 段 2～7 | V2V 接力 | **上一段尾帧图 URL**（tail_01～tail_06） | 视频 + 截取尾帧 → tail_02～tail_07，各上传得 URL 供下一段 |
| 段 8 | V2V 接力 | **尾帧图 URL**（tail_07） | 视频（无需再截尾帧） |

### 与下一段衔接

- 尾帧：上一段视频**最后一帧**截取后，用 `scripts/upload-image-url.sh` 上传得到 URL，作为下一段 `image_urls[0]`。
- 提示词：描述「从上一帧之后接下来发生的动作」，保留 same scene、consistent character、夏日海边光线一致。

## 三、Phase 目录结构与生图/生视频约定

- **比例**：所有生成素材（生图、生视频）统一 **16:9（长:宽）**。
- **生图与生视频是分开的步骤**：先按生图提示词生成首帧/尾帧图，再以该图 + 生视频提示词做图生视频；提示词分文件存放，不混在同一段文字里。
- **生图提示词须极尽细致**：场景、所有物件、动作、穿着、微表情与神态、氛围等都要写清楚，直到逼近提示词上限，这样后续生视频效果才好。可写：场景与地面细节（沙、贝壳、泡沫、脚印）、天空与光、人物服装（款式、颜色、质感）、站姿与间距、视线与微表情、头发与衣角被风吹动、光影与肤色、负面约束等。
- **严禁角色脸型与身材严重变形**：生图与生视频提示词中须明确禁止脸型变形、身材比例变形、五官错位、头身比例失调、肢体拉长或压缩，并强调角色脸型与身材须与参考图一致、不得扭曲变形。
- **生图只传人物图与首帧、不传环境图**：上传参考图时仅使用 refs/characters 下人物图及本段首帧（尾帧生图时），不传入 refs/scenes 环境图；提示词中须强调「人物在生图中具有绝对地位，必须严格按人物参考图还原脸型与身材，场景仅作氛围衬托」。
- **首帧与尾帧须根据故事情节而定**：首帧 = 该段视频开场第一秒画面，尾帧 = 该段结束最后一秒画面。**须把该时刻的故事情节写进生图提示词**（例如首帧写「两人刚到海边、刚站定望海、蕾娜刚开口『到啦—』的那一瞬」，尾帧写「两人并排看海、段尾定格、蕾娜即将迈步跑向海浪的衔接瞬间」），而不是只写静态画面；这样生成的图才适合作为该段真正首帧/尾帧。

每个 phase 目录内：

| 文件 | 用途 |
|------|------|
| `phaseXX.txt` | 本段情节说明、素材说明、台词、**给即梦生视频的整份提示词**（≤2000 字）、负面词等；仅用于**生视频**步骤。 |
| `phaseXX_img_first.txt` | **首帧生图**专用提示词（单独文件）；仅用于即梦**生图**步骤，生成本段第一帧。 |
| `phaseXX_img_last.txt` | **尾帧生图**专用提示词（单独文件）；尾帧=下一段首帧，故也须写详细提示词（与首帧同级），用于生图得到 tail_XX 或下一段首帧图。若本段仅从视频截取尾帧可不生图，但建议仍写满提示词以便衔接一致。 |

生成的素材（如 phase 视频、截取的尾帧图）与上述提示词文件均放在**对应 phase 目录**下。
