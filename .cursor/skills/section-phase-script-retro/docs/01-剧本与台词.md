# 一、剧本与台词（内容与节奏）

本主题合并：**台词密度与节奏**、**动作/神态/寡言人设**、**对话回应不跨 phase**。用于自检「写什么、怎么说、谁反应」。

---

## 1. 台词密度与节奏

### 问题简述

- 每段 15s 内多为「2 个时间块」，且每句字数/信息量接近；缺少「快段」（多句短台词、抢话、接话）与「慢段」（少台词、眼神/动作为主），没有呼吸感。
- **台词密度脱离人语速**：如 4 秒内只有 8 字、一句台词占 10 秒却只有十几个字等，观感机械、不像真人对话（**section03 对话重写复盘**）。

### 改进要点

1. **在总述/README 中规划节奏**：标出哪几段是「对话密集、节奏快」，哪几段是「台词少、留白多」。
2. **台词时间块要有差异**：快段多拆时间块、短句接话；慢段少台词、大块留白或整段仅 1～2 句。
3. **剧本台词块**：在 phase 的「本段剧本台词」里体现时间轴切分与每块字数句数，便于配音与提示词一致。
4. **可选**：分块 phase 设计时，块边界与台词时间块对齐。
5. **台词密度按人语速**（见下「1.1 台词密度与人语速」）。

### 1.1 台词密度与人语速（section03 对话重写经验）

**原则**：每个台词时间块内的**字数需符合真人语速**，避免「4 秒只说 8 个字」等机械分配。

**参考标准**（日语配音）：
- **1 分钟约 300～400 字**（业界常见目安）；换算为 **1 秒约 5～7 字**（约 6～7 字/秒较自然）。
- 例：5 秒块 ≈ 30～35 字；8 秒块 ≈ 48～56 字；10 秒块 ≈ 60～70 字。

**落实**：
- 在「本段剧本台词」中按时间块分配台词时，**先算该块秒数 × 约 6 字/秒**，再写相应长度的句子或句数；可多句短台词、接话，避免一大块时间只配一句极短台词。
- 在 00 或 README 中可注明「台词密度约 6～7 字/秒（日文）」作为本 section 约定；配音/TTS 时按此调整语速。

### 自检清单（台词）

- [ ] 至少有一段「对话密集」：时间块 ≥3，短句/接话多。
- [ ] 至少有一段「留白为主」：台词少，大段时间为动作/神态。
- [ ] 各段台词长度/句数有明显差异，不是每段都「两句均分 15s」。
- [ ] **每个台词时间块内字数符合人语速**（如日文约 6～7 字/秒），无「4 秒只有 8 字」等机械感。

## 2. 动作 / 神态 / 寡言人设

### 问题简述

- 情节与提示词多用概括句，缺少可被画面执行的**动作链与神态变化**；寡言/内敛若只写「少言」而未写**微反应**，成片易呈现一方活跃、另一方近乎静态。

### 改进要点

1. **情节说明**：补上动作顺序、视线、微表情（例：先抿嘴憋笑、视线瞟向他，再嘴角上扬、手背挡嘴）。
2. **提示词【本段】**：拆成多个可执行动作；加入表情变化（从什么到什么）；两人互动写清 **A 做 X → B 做 Y**；【本段动作与节奏】用时间块，不贴整张秒级表；秒级用线性表述、不另做表格。
3. **寡言/内敛人设**：在 00/README 明确寡言 = 少台词，**不意味着**无表情无反应；秒级表该角色每时间块须有对对手方的**具体微反应**（视线/喉结/呼吸/手指等）；【本段动作与节奏】显式写「对手做 X → 该角色做 Y」。
4. **寡言 ≠ 只会说「嗯」**：亲密双人场合下须有**有内容的短句**（如「うん、行こう」「やっとだな」），非仅应声。
5. **接话不复读**：接话用不同角度或自己的说法，不原样/近义复读对方刚说的词句。
6. **台词在提示词中的强化**：粘贴块内增加【本段对白（须体现口型）】按时间块列出谁在何时说什么；【本段动作与节奏】中嵌入口型/关键句台词（引号内）。

### 自检清单（动作/神态/寡言）

- [ ] 情节说明是否有动作顺序 + 视线/微表情。
- [ ] 提示词是否有「A 做 X、B 做 Y」的互动链，及至少一处表情变化。
- [ ] 寡言方秒级表每时间块是否有对对手方的具体回应；【本段动作与节奏】是否写出触发-反应链。
- [ ] 寡言方是否有有内容的短句，非仅「嗯」「ああ」；接话是否复读，若复读是否已改写法。
- [ ] 粘贴块是否有按时间块的本段对白；【本段动作与节奏】是否写出关键句台词（引号内）。

## 3. 对上一句的回应不放到下一 phase

**规则**：不要把**对上一句台词的回应**放到下一个 phase，否则 Seedance 无法在同一段内建立「说—应」关系，导致对话与口型/节奏处理不当。

### 原因

- 下一 phase 以**上一段尾帧**延续的是画面，不是「上一句台词」的上下文；若 B 在 phase N+1 开头说「嗯」接 A 在 N 末的「终于啊」，模型在 N+1 里看不到 A 的台词，易口型/节奏错位。

### 正确做法

- **同一组「陈述/问句 + 回应」尽量放在同一 phase**；若必须切段，则**把回应收在上一个 phase 的段尾**，下一 phase 从新话题或新动作开始。

### 自检清单

- [ ] 本 phase 开头是否有对**上一 phase 最后一句台词**的直接回应？若有，是否已把该回应挪到上段尾或把「上句+回应」放在同一 phase？
- [ ] 写新 phase 时是否扫过上段【本段剧本台词】最后一句，避免本段 0～4s 出现「接上句」的对话？

## 4. 对话内容具体化与作品背景（section03 对话重写经验）

**问题**：对话多为空泛感慨（如「能这样走着真好」「好舒服呢」），缺少**具体话题与作品/世界观挂钩**，观感像机器填空、不像有背景的真人聊天。

**改进要点**：

1. **避免空泛感慨**：少用仅表达情绪的笼统句（「真好呀」「好开心」）；改为**具体话题**：谁做了什么、提到什么人/事、对未来的具体打算（如「仪式之后再来」「蜜月来这里」）、对过去的具象回忆（如「以前没想过能活着站在这里」）。
2. **用作品/世界观设定写对话**：若为改编或同人，**先查资料**——原作结局、角色关系、名台词、角色愿望或经历（如「辛的愿望是带蕾娜去看海」、战友名字、战后/结婚前时间点）。把可用的设定写成**具体台词**（如提战友名、约定「下次大家一起来」、调侃「辛君的愿望不是吗」），而不是泛泛的「能这样在一起真好」。
3. **创作前查资料**：
   - **作品设定**：搜索原作剧情、角色关系、时间线、名场面或名句，列出可化用为台词的点。
   - **语速/字数参考**：搜索「日语 配音 1秒 字数」「动画 剧本 台词 密度」等，确认本 section 所用语言的大致字/秒，再分配时间块字数。
4. **寡言方也要有「有内容的短句」**：接话时带动机、认同、约定或本音（如「是你说想来的」「下次大家一起来吧」「是因为有你在」），而不是只答「嗯」「啊」。

**自检清单**：
- [ ] 本 section 对话是否有**具体话题**（人/事/约定/回忆），而非只有空泛感慨？
- [ ] 若为改编/同人，是否已**查过原作设定**并化用为可说的台词（愿望、战友名、时间点等）？
- [ ] 寡言方的短句是否**有内容**（动机/认同/约定/本音），而非仅「嗯」「啊」？
