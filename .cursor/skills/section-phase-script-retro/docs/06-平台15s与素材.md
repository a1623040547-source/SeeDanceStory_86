# 六、平台 15s 与素材（全能参考与配乐）

本主题合并：**全能参考 15s**、**配乐与 15s 封闭**。用于自检「平台 15s 限制下的参考与配乐」。

---

## 1. 全能参考视频上传（平台 15s 限制）

### 问题简述

- 使用**全能参考**时**不能只传上一段视频**：需要同时提供「整体环境/风格」与「下一段连续性」两类参考。平台参考视频总时长限制 **15 秒**以内；做法是拆成**两个独立视频**（7s + 8s），**分别**上传/使用，**不要**把 7s 和 8s 拼成一条 15s 再传。

### 正确做法

1. **两个独立视频，不拼接**
   - **7s 视频**（`ref_env_7s.mp4`）：从 **section 第一段（phase01）视频**中截取**前 7 秒**。用途：**整体环境/风格参考**（打光、场景、色调、氛围），供下一段生成时锁定与段 1 一致的整体观感。
   - **8s 视频**（`ref_continuity_8s.mp4`）：从**当前最新 phase** 视频中截取**后 8 秒**。用途：**下一段连续性参考**（V2V 衔接、动作/状态接续）。
   - 使用时：**分别**上传或分别选为「环境参考」与「连续性参考」，不要合并成一条 15s。

2. **特殊情形**：若第一段即为最新（当前只有 phase01 有视频），则 7s 与 8s 均从该段截取（前 7s 一份、后 8s 一份），仍是两个文件。

### 全能参考首段提示词（上传时填写）

```
为人物参考 ｜
为故事发生环境参考（环境需要适配人物画风）｜
为上一段视频尾帧参考，如果没有切换镜头过渡的话，需要用在这一段首帧｜
为第一段视频参考，让这一段视频与第一段画面风格保持一致
```

### 使用脚本

脚本位置：`.cursor/skills/section-phase-script-retro/scripts/build_ref_15s.py`，依赖 **ffmpeg**。输出放在 `{section}/phases/` 下：`ref_env_7s.mp4`、`ref_continuity_8s.mp4`。

```bash
python3 .cursor/skills/section-phase-script-retro/scripts/build_ref_15s.py --section section03
```

| 输出文件 | 来源 | 时长 | 作用 |
|----------|------|------|------|
| ref_env_7s.mp4 | section 第一段视频 | 前 7s | 整体环境/风格参考 |
| ref_continuity_8s.mp4 | 最新 phase 视频 | 后 8s | 下一段连续性参考 |

## 2. 配乐与 15s 封闭

### 问题简述

- Seedance 在 15s 内配乐时，若未在提示词中约束，易出现**音乐在 15s 结尾处突然截断**或**没有收束感**，影响单段观感与多段拼接连贯性。

### 改进要点

1. **在 phase 提示词中明确告知**：若本段使用配乐，配乐须在 15s 内**封闭**，即到 15s 结尾前有**明确的渐进终止**（fade out / 自然收束），避免段末硬切。
2. **强制要求：最后 1 秒不能有音乐**：视频**最后 1 秒内必须无配乐**（仅保留环境声或静音），避免段末 BGM 残留、与下一段拼接时突兀。
3. **否则尽量不用配乐**：若无法在提示词中落实「15s 内封闭 + 最后 1s 无音乐」，或该段更适合无配乐，则**尽可能不选用配乐**。

### 可写进提示词的建议用语

- 中文（推荐）：「本段若有配乐，须在 15 秒内封闭，结尾前有明确渐进终止，且**最后 1 秒不能有音乐**；否则本段不用配乐。」
- 英文可类比：If background music is used, it must fade out and end before the final second; the last 1 second must have no music. Otherwise do not use music for this segment.
