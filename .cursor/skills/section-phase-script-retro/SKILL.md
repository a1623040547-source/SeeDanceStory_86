# Section / Phase 剧本复盘与改进要点

**说明**：基于 API 生成连贯视频的 section04 已废案（见 `archive/section04/废案说明.md`）。根目录 section01～section04 已归档至 `archive/`（实验性质）；本 Skill 的复盘与改进要点仍适用于在 Seedance 平台手动生成的 section（含 story 下 section 与 archive 内参考）。

基于 section02 中期复盘结论，在**新建 section、撰写或改写 phase 剧本与提示词、或规划 phase 间衔接与镜头过渡**时应用，避免台词平均化、画面呆板与衔接方式单一。

## 一、何时使用本 Skill

**文档结构**：复盘要点已按框架合并为 `docs/00-框架与索引.md` 与 **01～07**（见下方「三、详细章节索引」）；原单题文档保留在 `docs/archive/` 供按题查阅。

- 用户要求**新建 section** 或**写/改 phase 剧本**时：在按《剧本与视频生成流程》搭好结构后，用本 Skill 自检；入口见 [docs/07-与剧本流程衔接.md](docs/07-与剧本流程衔接.md)，自检按 [docs/00-框架与索引.md](docs/00-框架与索引.md) 分层（01→…→06）。
- 用户做**剧本复盘**、**中期复盘**、**下一 section 改进**时：引用本 Skill 的改进要点与「四、执行清单」；按问题类型对应 [docs/01-剧本与台词.md](docs/01-剧本与台词.md)～[docs/06-平台15s与素材.md](docs/06-平台15s与素材.md)。
- 用户反馈**生成画面人物活动性低、互动呆板**、**寡言/内敛角色在成片中像无反应**或**对话没有快慢节奏**时：对照 [docs/01-剧本与台词.md](docs/01-剧本与台词.md)（台词密度、动作神态、寡言微反应、对话不跨 phase）。
- 用户反馈**对话像机器、不自然**、**台词太稀疏**（如 4 秒只有几个字）或**对话空泛没内容**时：对照 [docs/01-剧本与台词.md](docs/01-剧本与台词.md) 1.1（台词密度与人语速）、第 4 节（对话内容具体化与作品背景）；创作前查**作品设定**与**语速/字数参考**。
- 用户讨论**phase 间承接、镜头过渡、首尾帧以外衔接**、**环环相扣**、**V2V 切镜与 phase 时长**时：引用 [docs/02-Phase衔接与切镜.md](docs/02-Phase衔接与切镜.md)。
- 用户反馈**画面随 phase 漂移（如变冷光、下雨变下雪）**、**人景位置错乱**、**门/环境描写过笼统**时：引用 [docs/03-场景环境与空间.md](docs/03-场景环境与空间.md)。
- 用户要求**提示词更专业、运镜/镜头/程度说明**、**画面人数**、**走路/踩水与位移**时：引用 [docs/04-运镜与画面约束.md](docs/04-运镜与画面约束.md)。
- 用户在做**全能参考上传视频**或**配乐在 15s 内截断**时：引用 [docs/06-平台15s与素材.md](docs/06-平台15s与素材.md)。
- 用户反馈**穿着跨段不一致**、**物品/动作前段已完成后段又写**、**模糊表述「A 或 B」导致错乱**时：引用 [docs/05-跨Phase一致性.md](docs/05-跨Phase一致性.md)。

## 二、复盘结论（section02 / section03 中期；section03 对话重写）

以下按**框架层级**归类；详细改进要点与自检见 `docs/00-框架与索引.md` 及对应文档（01～06）。**section03 对话重写**补充：台词密度按人语速、对话内容具体化与作品背景、创作前查资料。

| 层级 | 问题与表现 | 改进方向概要 | 文档 |
|------|------------|--------------|------|
| **一、内容与节奏** | 台词密度相近、缺快慢段；**密度脱离人语速**（如 4 秒只有 8 字）；**空泛感慨多、缺具体话题与作品背景**；动作/神态单薄；寡言像无反应或只会「嗯」；对上一句回应放到下一 phase | 快段/慢段；**台词密度按人语速**（如日文约 6～7 字/秒）；**对话具体化+查作品设定与语速参考**；动作链+微反应+有内容短句；陈述+回应同 phase 或回应收上段尾 | [01-剧本与台词](docs/01-剧本与台词.md) |
| **二、Phase 结构** | 仅依赖首尾帧 V2V；段间观感断裂；V2V 连续画质下降；每段都 15s 导致节奏脱节 | 过渡方案表、环环相扣、最多每 4 phase 切镜、4～15s 按内容定时长 | [02-Phase衔接与切镜](docs/02-Phase衔接与切镜.md) |
| **三、场景与空间** | 环境/打光随 phase 漂移；人景位置错乱；门用手开失败；环境描写笼统 | 环境统筹+第一段环境参考图；提示词写清人景位置+自动门+细致场景描述 | [03-场景环境与空间](docs/03-场景环境与空间.md) |
| **四、运镜与画面** | 缺景别/运镜/程度；同一人在画面出现两次；走路/踩水变原地踏步 | 景别+运镜类型与程度；明确画面人数；区分站定与有位移走路 | [04-运镜与画面约束](docs/04-运镜与画面约束.md) |
| **五、跨段一致性** | 穿着跨 phase 不一致；物品/动作前段已完成后段又写；模糊「A 或 B」导致错乱 | 穿着锚定+ref 与阶段一致；物品/动作时间线；明确指向一种 | [05-跨Phase一致性](docs/05-跨Phase一致性.md) |
| **六、平台与素材** | 全能参考只传上一段；配乐 15s 末硬切 | 7s 环境+8s 连续性两段独立视频；配乐 15s 内封闭、最后 1s 无音乐 | [06-平台15s与素材](docs/06-平台15s与素材.md) |

## 四、执行清单（新建或改写 section 时）

- [ ] **节奏规划**：在 README 或 00 中标出快段/慢段，并落实到各段台词时间块。
- [ ] **台词密度**：至少 1 段密集、1 段留白，各段句数/字数有差异；**每个台词时间块内字数按人语速**（如日语约 6～7 字/秒），避免「4 秒只有 8 字」等机械感；见 [docs/01-剧本与台词.md](docs/01-剧本与台词.md) 1.1。
- [ ] **对话内容具体化**：避免仅空泛感慨；用**具体话题**（人/事/约定/回忆）与**作品/世界观设定**（原著愿望、战友名、时间点等）；若为改编/同人，**创作前查作品设定**并化用为台词；见 [docs/01-剧本与台词.md](docs/01-剧本与台词.md) 第 4 节。
- [ ] **情节说明**：每段含动作顺序 + 视线或微表情，不只一句概括。
- [ ] **提示词本段**：每段含可执行动作链与 A→B 互动，至少一处表情/神态变化描述。
- [ ] **寡言/内敛角色**：若存在此类人设，00 或 README 已澄清「寡言≠无反应」且**寡言≠只会说嗯**（亲密双人场合下须有有内容的短句）；秒级表该角色列每时间块有对对手方的具体微反应；【本段动作与节奏】显式写「对手做 X→该角色做 Y」的触发-反应链。
- [ ] **phase 衔接与过渡**：在 00 或 README 中为相邻 phase 选定或标注过渡方案（首尾帧 / 动作切点 / 景别/黑场/节奏等）；各 phase 边界写清段尾/段头动作或景别。
- [ ] **环环相扣**：每段情节说明中部分提及「承接上段末尾…」与「本段收束/下段将…」；提示词（若允许）可带一句前后衔接说明。
- [ ] **环境/氛围/打光统筹**：在 00 与 refs 中约定「第一段环境氛围参考图」；各段提示词中明确环境/光线/窗外天气与段 1 一致；段 2～12 上传该参考图以锁定氛围。
- [ ] **人与场景位置关系**：每段提示词中写出人物在场景中的具体位置（如窗左/沙发右/两人相对）；有运镜时显式约束运镜过程中人景相对位置不变；与 00 或第一段约定的空间布局一致；写下一段前通过**上一段尾帧**观察实际人景位置，据此调整本段提示词以保持空间延续。
- [ ] **有运动处写清怎样运动**：凡有泼水、撩水、回击、挡、躲、扶等明显动作的段落，须在情节说明与提示词中写清身体怎样动（如弯腰/蹲低捧水泼向、手臂从身侧撩水向对方、抬手挡脸、侧身躲），不能仅用「泼水」「回击」等概括词；见 [docs/03-场景环境与空间.md](docs/03-场景环境与空间.md)、[docs/04-运镜与画面约束.md](docs/04-运镜与画面约束.md)。
- [ ] **门与环境设计**：涉及门进出时一律用**自动门（横向滑开/滑闭）**，不写推门/拉门/转门把；refs/scenes 或 00 中是否有**细致场景描述**（空间布局、门/窗/家具位置与样式、材质光线、动线），达到可据此画出一致画面的程度；见 [docs/03-场景环境与空间.md](docs/03-场景环境与空间.md)。
- [ ] **物品关系与情节连贯**：创作时考虑**位置关系、情节关系、物品关系**；关键物品/动作（行李、包、帽子等）在 section 内有简单时间线或标注；写每段前检查前面各 phase 是否已完成该动作或改变该物品状态，避免本段重复（如 phase02 已放好行李则 phase04 不再写「放好行李」）；见 [docs/05-跨Phase一致性.md](docs/05-跨Phase一致性.md)。
- [ ] **运镜/镜头/程度**：每段提示词写出景别与运镜类型+程度；00 或 README 可做本 section 运镜约定。
- [ ] **提示词块格式**：粘贴到 Seedance 的块按《phase-prompt-format》五段式；【本段动作与节奏】用时间块而非逐秒 15 行。
- [ ] **台词/对白**：有台词的段在粘贴块内包含「本段对白（须体现口型）」按时间块列出谁说什么，且【本段动作与节奏】中嵌入关键句台词（引号内）。
- [ ] **穿着状态**：若有明显穿着变化（如脱外套），脱之后各 phase 的提示词/情节说明中是否有男女主穿着状态的明确一句；ref 图是否与剧情阶段一致或已在提示词中强调。
- [ ] **全能参考**：是否拆成**两个独立视频**（7s 环境 + 8s 连续性）分别上传/使用，而非拼成一条 15s。
- [ ] **配乐**：若某段使用配乐，提示词中是否已写明「配乐在 15s 内封闭、结尾前有明确渐进终止，且最后 1 秒不能有音乐」；否则该段尽量不用配乐。
- [ ] **对话不跨 phase**：本 phase 开头是否出现对**上一 phase 最后一句台词**的直接回应（如「嗯」接上段）？若有，是否已把该回应挪到上段尾或把「上句+回应」放在同一 phase；见 [docs/01-剧本与台词.md](docs/01-剧本与台词.md)。
- [ ] **无模糊表述**：情节说明/物品/提示词中是否出现「A 或 B」「A/ B」类写法？若有，是否已改为唯一确定项并与 00、其他 phase 一致；见 [docs/05-跨Phase一致性.md](docs/05-跨Phase一致性.md)。
- [ ] **画面人数明确**：Seedance 提示词中是否已写明本段**画面内人数**（如「仅两人、无其他人物」），避免同一人重复出现；见 [docs/04-运镜与画面约束.md](docs/04-运镜与画面约束.md)。
- [ ] **走路与位移**：若本段有「走」「踩水」「玩水时移动」，是否已区分站定与有位移？站定写「位置基本不变、不向前行走、避免原地踏步感」；有位移走路写「沿某方向前进、相对背景有位移、不是原地踏步」；见 [docs/04-运镜与画面约束.md](docs/04-运镜与画面约束.md)。
- [ ] **V2V 切镜边界**：本 section 是否在 phase 4、8、12… 等边界做了「切镜」规划（该段起不再用上段尾帧 V2V）？这些交界处的上一段尾是否设计为**画面较平稳**的收束；见 [docs/02-Phase衔接与切镜.md](docs/02-Phase衔接与切镜.md)。
- [ ] **Phase 时长**：各 phase 是否在**平台 4～15s 范围内**按对话密度/内容密度设定时长（00/README 可标建议时长或短/中/长），而非默认每段 15s；见 [docs/02-Phase衔接与切镜.md](docs/02-Phase衔接与切镜.md)。
