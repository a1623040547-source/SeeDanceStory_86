# Section / Phase 剧本复盘与改进要点

基于 section02 中期复盘结论，在**新建 section、撰写或改写 phase 剧本与提示词、或规划 phase 间衔接与镜头过渡**时应用，避免台词平均化、画面呆板与衔接方式单一。

---

## 一、何时使用本 Skill

- 用户要求**新建 section** 或**写/改 phase 剧本**时：在按《剧本与视频生成流程》搭好结构后，用本 Skill 自检台词节奏与动作描写。
- 用户做**剧本复盘**、**中期复盘**、**下一 section 改进**时：引用本 Skill 的改进要点与检查清单。
- 用户反馈**生成画面人物活动性低、互动呆板**或**对话没有快慢节奏**时：对照本 Skill 加强台词密度与动作/神态描写。
- 用户讨论**phase 间承接、镜头过渡、首尾帧以外衔接**时：引用本 Skill 的镜头过渡方案与剧本落点。
- 用户强调**剧情/人物衔接自然、phase 环环相扣**时：引用本 Skill 第六节「当前 phase 内提及前后段」。
- 用户反馈**画面随 phase 漂移（如变冷光、下雨变下雪、画面变花）**或要求**统筹全片环境/氛围**时：引用本 Skill 第九节「环境/氛围/打光统筹」与 refs 中的环境氛围参考图。
- 用户要求**提示词更专业、符合即梦/视频生成平台规范**或**补充运镜/镜头/程度说明**时：引用本 Skill 第十节「运镜/镜头/程度说明」。

## 二、复盘结论（section02 中期）

| 问题 | 表现 | 改进方向 |
|------|------|----------|
| **台词密度与节奏** | 每段台词长度相近，缺少故事中对话的快慢节奏感 | 设计「快段」与「慢段」：有的段台词多、短句多、节奏快；有的段台词少、留白多、节奏慢 |
| **动作/神态/细节描写单薄** | 提示词与情节说明对动作、神态、细节写得少且笼统，导致生成画面人物活动性低、互动呆板 | 在情节说明与 Seedance 提示词中补充：具体肢体动作、表情变化、视线与微表情、两人互动节奏（谁先动、谁反应） |
| **phase 间承接方式单一** | 仅依赖「上一段尾帧 → 下一段首帧」V2V 衔接，缺少在剧本阶段的过渡设计 | **在创建剧本时**就以 15s 为一 phase，为每对相邻 phase 预先提出多种镜头过渡方案（首尾帧 / 动作切点 / 景别变化 / 黑场或淡入淡出 / 情绪节奏切点 / 台词桥梁），并写入 00 或 README 及 phase 边界 |
| **phase 间剧情/人物衔接不自然** | 每段只写本段内容，不点明与上一段尾部、下一段首部的故事与状态衔接，导致观感断裂、人物状态跳变 | **在当前 phase** 内部分提及「上一 phase 尾部发生了什么」与「下一 phase 首部将发生什么」，形成 phase 之间环环相扣；落点在情节说明与提示词 |
| **环境/氛围/打光随 phase 漂移** | 虽保证了上一 phase 向下一 phase 的衔接，但画面与整体环境、打光未统筹：前段暖光+窗外下雨，后段逐渐变为下雪或冷光，画面变花 | **在提示词中统筹全片环境/氛围**；额外提供**第一段环境氛围参考图**到 refs，段 2～12 生成时作为环境/打光/窗外天气参考，锁定与段 1 一致 |

## 三、台词密度与节奏

### 问题简述

- 每段 15s 内多为「2 个时间块」（如 0～7s 一句、8～15s 一句），且每句字数/信息量接近。
- 整篇没有「连珠炮」式的快段（多句短台词、抢话、接话）也没有「留白」式的慢段（少台词、眼神/动作为主），缺少故事中的呼吸感。

### 改进要点

1. **在总述/README 中规划节奏**：标出哪几段是「对话密集、节奏快」（如抢零食、拌嘴），哪几段是「台词少、留白多」（如递茶、盖毯子、安静收尾）。
2. **台词时间块要有差异**：
   - **快段**：多拆时间块（如 0～3s / 3～6s / 6～10s / 10～15s），短句、接话、语气词、笑声。
   - **慢段**：少台词、大块留白（如 0～12s 只有动作/神态，12～15s 一句收束），或整段仅 1～2 句。
3. **剧本台词块**：在 phase 的「本段剧本台词」里体现上述节奏（时间轴切分、每块字数与句数），便于配音/TTS 与提示词中的 Dialogue 一致。

### 自检清单（台词）

- [ ] 至少有一段「对话密集」：时间块 ≥3，短句/接话/语气词多。
- [ ] 至少有一段「留白为主」：台词少，大段时间为动作/神态。
- [ ] 各段台词长度/句数有明显差异，不是每段都是「两句均分 15s」。

## 四、动作 / 神态 / 细节描写

### 问题简述

- 情节说明与提示词多用概括句（如「青年抱臂表情严肃，少女憋笑偷看他」），缺少**可被画面执行的**动作链与神态变化。
- 结果：模型难以生成细腻的肢体与表情，人物活动性低、互动显得呆板。

### 改进要点

1. **情节说明**：不只写「谁做了什么」，补上**动作顺序、视线、微表情**。  
   例：不说「蕾娜憋笑偷看」，而写「蕾娜先抿嘴憋笑、视线瞟向他，再忍不住嘴角上扬、用手背挡一下嘴」。
2. **Seedance 提示词（本段部分）**：  
   - 把「一句话概括」拆成**多个可执行动作**：谁先动、谁后动、手/身体/视线如何配合。  
   - 加入**表情变化**：从什么表情到什么表情（如 from slight frown to holding back a smile）。  
   - 两人互动时写清**触发与反应**：A 做 X → B 做 Y（如 young man crosses arms → young woman glances at him then bites her lip to keep from laughing）。
3. **与平台限制平衡**：若整份粘贴且平台对英文敏感，提示词用中文时同样要写细动作与神态，避免只写「抱臂」「憋笑」等笼统词。

### 自检清单（动作/神态）

- [ ] 本段情节说明里是否有「动作顺序 + 视线/微表情」而不只有一句概括。
- [ ] 本段提示词里是否有「A 做 X、B 做 Y」的互动链，而不是仅「A 和 B 在做什么」。
- [ ] 是否包含至少一处「表情变化」或「从…到…」的神态描述。

## 五、phase 间承接与镜头过渡方案

以 15s 为一 phase **构建剧本时**，phase 间承接除「上一段尾帧 → 下一段首帧」的 V2V 方案外，**在创建剧本阶段**就为每对相邻 phase 提出并标注多种可选的镜头过渡方案；这样生成与剪辑阶段有据可依，避免只依赖首尾帧一种方式、或成片后再硬切。

### 可选方案（创建剧本时即可标注）

| 方案 | 说明 | 在剧本中的落点 |
|------|------|----------------|
| **首尾帧衔接** | 上一段最后一帧 = 下一段视觉起点，V2V 连续（当前默认） | 00/本段素材：尾帧图；提示词：same scene, consistent character |
| **动作切点** | 在动作自然断点处分段（如手放下后、转身完成时） | 情节说明/提示词：段尾以「手放杯结束」、段头以「转身后开始」等明确起止动作 |
| **景别/镜头变化** | 段界处换景别（如中景→特写、或切到空镜再回人） | README/00：标出「段 N 以中景开场」「段 N 尾 2s 为茶几空镜」；提示词本段写清景别 |
| **黑场/淡入淡出** | 段与段之间插入短黑场或淡出淡入 | 在 README 剪辑时间轴或 phase 备注中写「段 M 与段 N 之间可加 0.5s 黑场」或「段 N 前淡入」 |
| **情绪/节奏切点** | 用节奏或情绪转折作为软过渡（紧张段后接缓和段） | 00/README：标出「段 5→6 节奏由紧到松」「段 8 为情绪转折」；段头段尾情节说明写清情绪 |
| **声音/台词桥梁** | 用台词或环境声跨段（如「那就这样吧」→黑场→「第二天」） | 剧本台词：段尾一句收束句，段头一句起句；或标注「此处可接环境声/BGM 过渡」 |

### 使用要点

1. **在 00 或 README 中做 phase 衔接总表**：列出每对相邻 phase（如 1→2、2→3…）采用的过渡方式（首尾帧 / 动作切点 / 景别变化 / 黑场等），便于生成与剪辑一致。
2. **在 phase 内写清边界**：本段情节说明或提示词末尾注明「段尾保留动作/静帧/景别」；下一段注明「段头从何动作/景别开始」。
3. **与平台能力匹配**：若当前仅支持首尾帧 V2V，其他方案可在剪辑阶段实现，但剧本里先标好，避免成片再硬切。

详细示例见 [reference.md](reference.md) 第 4 节。

## 六、当前 phase 内提及前后段（环环相扣）

为保证剧情与人物形象衔接自然，**当前 phase** 的剧本与提示词应**部分提及**上一 phase 的尾部故事与下一 phase 的首部故事，使段与段之间环环相扣，避免「每段只写本段」导致的观感断裂或人物状态跳变。

### 改进要点

1. **【本段情节说明】**  
   - 段头：用一两句点明「承接上段末尾的什么状态/动作」（如「承接上段蕾娜整盘端走后」）。  
   - 段尾：用一两句点明「本段收束在什么状态，为下段什么开头做铺垫」（如「以辛恩板脸瞥她结束，下段蕾娜憋笑破功」）。  
   - 不必长，目的是让读者与生成方明确「从哪接到哪」。

2. **Seedance 提示词（本段部分）**  
   - 开头可带一句「承接」：如「Continuing from previous segment: young woman has just pulled the plate to her side. This segment: ...」或中文「承接上段少女将盘子挪向自己后。本段：…」。  
   - 结尾可带一句「过渡到下一段」：如「Segment ends with young man’s stern look; next segment will open with young woman breaking into laughter.」或「本段以青年板脸结束，下段由少女笑场开场。」  
   - 若平台字数敏感，可仅在情节说明中写清，提示词保持简洁，但情节说明的「前后提及」必做。

3. **与「镜头过渡」配合**：上一段尾部/下一段首部的提及，与 phase 衔接总表中的动作切点、景别、情绪切点一致，避免说法矛盾。

### 自检清单（环环相扣）

- [ ] 本段情节说明是否包含「承接上段…」或「上段末尾…」的表述（段 2～N）。
- [ ] 本段情节说明是否包含「本段以…结束」「下段将…」或「为下段…做铺垫」的表述（段 1～N-1）。
- [ ] 提示词（若允许）是否在段头或段尾有一句前后衔接说明。

详细示例见 [reference.md](reference.md) 第 5 节。

## 八、执行清单（新建或改写 section 时）

- [ ] **节奏规划**：在 README 或 00 中标出快段/慢段，并落实到各段台词时间块。
- [ ] **台词密度**：至少 1 段密集、1 段留白，各段句数/字数有差异。
- [ ] **情节说明**：每段含动作顺序 + 视线或微表情，不只一句概括。
- [ ] **提示词本段**：每段含可执行动作链与 A→B 互动，至少一处表情/神态变化描述。
- [ ] **phase 衔接与过渡**：在 00 或 README 中为相邻 phase 选定或标注过渡方案（首尾帧 / 动作切点 / 景别/黑场/节奏等）；各 phase 边界写清段尾/段头动作或景别。
- [ ] **环环相扣**：每段情节说明中部分提及「承接上段末尾…」与「本段收束/下段将…」；提示词（若允许）可带一句前后衔接说明。
- [ ] **环境/氛围/打光统筹**：在 00 与 refs 中约定「第一段环境氛围参考图」；各段提示词中明确环境/光线/窗外天气与段 1 一致（如 warm lighting, rainy window）；段 2～12 上传该参考图以锁定氛围。
- [ ] **运镜/镜头/程度**：每段提示词写出景别（特写/近景/中景/全景/远景）与运镜类型+程度（固定机位/缓慢推拉等）；00 或 README 可做本 section 运镜约定。

## 九、环境/氛围/打光统筹

### 问题简述（section02 后期复盘）

- 虽保证了**上一 phase 尾帧 → 下一 phase 首帧**的 V2V 衔接，但**整体环境、打光、窗外天气**随 phase 往后发生漂移。
- 表现：第一段时环境光较暖、窗外为下雨，越到后面越易变为下雪或冷光，画面越来越花，与段 1 确立的氛围不一致。

### 改进要点

1. **在提示词中统筹全片环境/氛围**
   - 在**总述/规则**与**每段提示词**中明确写出本 section 的固定环境设定，例如：`warm indoor lighting, rainy window (not snow), same cozy living room atmosphere throughout`。
   - 每段本段描述里可重复一句环境锚定：如「Same warm lighting and rainy window as segment 1; no snow, no cold blue tint.」

2. **提供「第一段环境氛围参考图」到 refs**
   - **来源**：从段 1 生成视频中选取一帧能代表「暖光、雨窗、室内氛围」的画面（或直接用段 1 首图/尾帧），保存为环境氛围参考图。
   - **放置**：放入 `refs/env_atmosphere/`（或 `refs/scenes/` 并单独说明），命名如 `section02_env_ref.png`。
   - **使用**：段 2～12 生成时，除上传「上一段尾帧」外，若平台支持多图，**额外上传该环境氛围参考图**作为「环境/场景/打光」参考，使模型在角色与动作接力的同时，保持与段 1 一致的光线、窗外天气与整体色调。

3. **在 00 / 素材说明中写清**
   - 在「素材类型与衔接规则」中增加**环境氛围参考图**一类，说明其用途与使用段（段 2～12）。
   - 在「每段使用的素材一览」中注明：段 2～12 除尾帧外，建议（或必须）使用环境氛围参考图。

## 十、运镜 / 镜头 / 程度说明（即梦向）

### 问题简述（section02 进一步复盘）

- phase 提示词中**缺少具体的运镜效果、镜头说明、程度说明**，仅用「镜头稳定」「治愈氛围」等笼统表述，不够专业，不利于即梦/Seedance 等平台精准控效。
- 即梦等平台的推荐结构为：**主体动态 + 运镜类型（速度/角度）+ 场景变化 + 风格**；并支持在提示词中**绑定运镜、景别、首尾帧**等，实现可控效果。

### 改进要点

1. **每段提示词中明确写出「景别」**
   - 用标准景别词：**特写（面部/手部/物品）**、**近景（胸部以上）**、**中景（膝上/半身）**、**全景（全身）**、**远景（环境/空间）**。
   - 避免只写「两人在客厅」而写成「中景，双人半身，沙发与茶几在画内」或「特写，青年手部递茶杯」。

2. **每段提示词中明确写出「运镜类型」与「程度」**
   - **固定机位**：明确写「固定机位」「无运镜」「static camera」，适合对话、安静段落。
   - **推/拉/摇/移**：写清类型与程度，例如：
     - 推镜头：缓慢推进、轻微推进、从全景推到中景；
     - 拉镜头：缓慢拉远、从近景拉至室内全景；
     - 跟拍：轻微跟拍、肩部跟拍；
     - 摇：缓慢横摇、从 A 摇向 B。
   - **程度词**：缓慢、轻微、微微、逐渐、匀速，避免「镜头拉远」而无速度/幅度。

3. **与官方示例对齐（火山引擎/即梦文档）**
   - 介绍细节用**特写**，展现环境用**全景**，欢乐/互动可用**跟拍或轻微运镜**。
   - 首尾帧控制：若本段有明确运镜，可在提示词中写「首帧：…；运镜：…；尾帧：…」便于平台解析。

4. **在 00 或 README 中做「本 section 运镜约定」**
   - 列出本片统一约定：如「室内日常以固定机位、中景为主」「段 N 为特写」「段 M 尾 5s 缓慢拉远」等，各 phase 提示词与之一致。
