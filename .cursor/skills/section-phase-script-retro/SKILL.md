# Section / Phase 剧本复盘与改进要点

**说明**：基于 API 生成连贯视频的 section04 已废案（见 `section04/废案说明.md`）。本 Skill 的复盘与改进要点仍适用于 **section01/02/03** 等在 Seedance 平台手动生成的 section。

基于 section02 中期复盘结论，在**新建 section、撰写或改写 phase 剧本与提示词、或规划 phase 间衔接与镜头过渡**时应用，避免台词平均化、画面呆板与衔接方式单一。

---

## 一、何时使用本 Skill

- 用户要求**新建 section** 或**写/改 phase 剧本**时：在按《剧本与视频生成流程》搭好结构后，用本 Skill 自检台词节奏与动作描写。
- 用户做**剧本复盘**、**中期复盘**、**下一 section 改进**时：引用本 Skill 的改进要点与检查清单。
- 用户反馈**生成画面人物活动性低、互动呆板**、**寡言/内敛角色在成片中像无反应**或**对话没有快慢节奏**时：对照本 Skill 加强台词密度与动作/神态描写，以及 [docs/04-动作神态与寡言人设.md](docs/04-动作神态与寡言人设.md)。
- 用户讨论**phase 间承接、镜头过渡、首尾帧以外衔接**时：引用 [docs/05-镜头过渡.md](docs/05-镜头过渡.md)。
- 用户强调**剧情/人物衔接自然、phase 环环相扣**时：引用 [docs/06-环环相扣.md](docs/06-环环相扣.md)。
- 用户反馈**画面随 phase 漂移（如变冷光、下雨变下雪、画面变花）**或要求**统筹全片环境/氛围**时：引用 [docs/09-环境氛围打光.md](docs/09-环境氛围打光.md) 与 refs 中的环境氛围参考图。
- 用户要求**提示词更专业、符合即梦/视频生成平台规范**或**补充运镜/镜头/程度说明**时：引用 [docs/10-运镜镜头程度.md](docs/10-运镜镜头程度.md)。
- 用户在做**全能参考上传视频**或**平台 15s 视频参考**时：引用 [docs/11-全能参考15s.md](docs/11-全能参考15s.md)。
- 用户反馈**剧情中有穿着变化（如脱外套给女主）但下一段角色又穿回**、或讨论**人物形象参考图导致穿着与剧情不符**时：引用 [docs/12-穿着服装状态.md](docs/12-穿着服装状态.md)。
- 用户讨论**配乐在 15s 内截断、段末无收束**或要求**在提示词中约束 Seedance 配乐**时：引用 [docs/13-配乐与15s封闭.md](docs/13-配乐与15s封闭.md)。
- 用户反馈**运镜时人物与场景位置关系错乱**、或要求**在提示词中固定人景位置**时：引用 [docs/14-人与场景位置关系.md](docs/14-人与场景位置关系.md)。
- 用户反馈**某段开头对上一句的回应导致 Seedance 对话/口型处理不当**、或要求**写 phase 时避免把对上一句的回复放到下一段**时：引用 [docs/17-对话回应不跨phase.md](docs/17-对话回应不跨phase.md)。
- 用户要求**剧情设计不写「A 或 B」、须明确指向**或反馈**因「帽或包」等模糊描述导致生成错乱**时：引用 [docs/18-禁止模糊表述.md](docs/18-禁止模糊表述.md)。
- 用户反馈**同一人在画面中出现两次**或要求**在提示词中明确画面人数**时：引用 [docs/19-画面人数明确.md](docs/19-画面人数明确.md)。
- 用户反馈**某物品/动作在前段已完成、后段又重复**（如「phase02 已放行李、phase04 又写放行李」）、或要求**剧本创作时考虑位置/情节/物品关系**时：引用 [docs/15-物品关系与情节连贯.md](docs/15-物品关系与情节连贯.md)。

## 二、复盘结论（section02 / section03 中期）

| 问题 | 表现 | 改进方向 |
|------|------|----------|
| **台词密度与节奏** | 每段台词长度相近，缺少快慢节奏感 | 设计「快段」与「慢段」；详见 [docs/03-台词与节奏.md](docs/03-台词与节奏.md) |
| **动作/神态/细节描写单薄** | 提示词笼统，人物活动性低、互动呆板 | 情节说明与提示词补充具体动作、表情、视线、A→B 互动；详见 [docs/04-动作神态与寡言人设.md](docs/04-动作神态与寡言人设.md) |
| **phase 间承接方式单一** | 仅依赖首尾帧 V2V，缺少剧本阶段过渡设计 | 创建剧本时为每对相邻 phase 标注过渡方案（首尾帧/动作切点/景别/黑场/节奏）；详见 [docs/05-镜头过渡.md](docs/05-镜头过渡.md) |
| **phase 间剧情/人物衔接不自然** | 每段只写本段，观感断裂、状态跳变 | 当前 phase 内提及「承接上段…」「本段以…结束、下段将…」；详见 [docs/06-环环相扣.md](docs/06-环环相扣.md) |
| **环境/氛围/打光随 phase 漂移** | 前段暖光+雨窗，后段变冷光/下雪、画面变花 | 提示词统筹环境；提供第一段环境氛围参考图，段 2～12 使用；详见 [docs/09-环境氛围打光.md](docs/09-环境氛围打光.md) |
| **寡言/内敛角色无微反应**（section03） | 成片一方活跃、另一方近乎静态 | 寡言≠无反应；秒级表与提示词写清触发-反应链；详见 [docs/04-动作神态与寡言人设.md](docs/04-动作神态与寡言人设.md) |
| **寡言≠只会说「嗯」**（section01 海边度假） | 男主每句都是「嗯」「ああ」，观感像不会说话 | 寡言=少台词，≠只会应声；亲密双人场合下须有有内容的短句（接话、认同、带情境份量）；详见 [docs/04-动作神态与寡言人设.md](docs/04-动作神态与寡言人设.md) §寡言≠只会说「嗯」 |
| **全能参考不能只传上一段**（section03） | 需环境+连续性两类参考；拆成两个视频分别用 | 一个 7s（第一段前 7s 环境）+ 一个 8s（最新段后 8s 连续性），**两个独立视频**分别上传，不拼接；详见 [docs/11-全能参考15s.md](docs/11-全能参考15s.md) |
| **穿着/服装状态跨 phase 不一致**（section03） | 脱外套给女主后下一段男主又穿回；ref 图含外套易被合成进去 | 脱外套之后各 phase 显式写穿着状态；可准备「脱外套后」ref 或提示词强调「无外套」；详见 [docs/12-穿着服装状态.md](docs/12-穿着服装状态.md) |
| **配乐在 15s 末截断**（section03） | 配乐在段末硬切、无渐进终止 | 在 phase 提示词中写明：配乐须在 15s 内封闭、结尾前有明确渐进终止，且**最后 1 秒不能有音乐**；否则尽量不用配乐；详见 [docs/13-配乐与15s封闭.md](docs/13-配乐与15s封闭.md) |
| **运镜时人与场景位置关系错乱** | Seedance 对空间位置理解有限，运镜后人景相对位置易漂移 | 在 phase 提示词中明确人物在场景中的位置（如窗左/沙发右）；与初始化环境/空间布局一致；运镜时显式约束相对位置不变；详见 [docs/14-人与场景位置关系.md](docs/14-人与场景位置关系.md) |
| **跨 phase 物品/动作重复或矛盾**（section01） | 前段已完成的动作（如放好行李）在后段再次出现，情节逻辑断裂 | 创作时考虑**位置关系、情节关系、物品关系**；建立关键物品/动作时间线；写每段前检查前面各 phase 是否已涉及该物品/动作；详见 [docs/15-物品关系与情节连贯.md](docs/15-物品关系与情节连贯.md) |
| **门与开门动作**（中期复盘） | 无明确提示时人物无法正确执行用手开门（推/拉/转门把） | 环境中涉及进出时一律用**自动门（横向滑动进出）**；提示词写「门横向滑开、人物走出」等，不写推门/拉门；详见 [docs/16-环境与门的设计.md](docs/16-环境与门的设计.md) |
| **环境描写过于笼统** | 仅一句概括导致图生/各段场景细节不一致、人景关系难延续 | 像小说家一样把场景细节交代清楚；refs/scenes 与 00 中应有完整空间布局、家具位置与样式、材质光线、动线；详见 [docs/16-环境与门的设计.md](docs/16-环境与门的设计.md) |
| **对上一句的回应放到下一 phase**（section01 phase06/07 复盘） | 上段末 A 说「终于啊」，下段头 B 说「嗯」接住，Seedance 无法在同一段内建立说—应关系，对话与口型易错位 | **同一组「陈述+回应」放在同一 phase**，或把回应收在上段尾；下段从新话题/新动作开始；详见 [docs/17-对话回应不跨phase.md](docs/17-对话回应不跨phase.md) |
| **模糊表述「A 或 B」**（section01 phase05/06 复盘） | 「持帽或包」「帽子或小包」等导致生成时前后不一致、画面错乱 | 任何位置**明确指向**一种（是帽子就写帽子）；详见 [docs/18-禁止模糊表述.md](docs/18-禁止模糊表述.md) |
| **画面人数未明确** | 未写「画面内几人」时，Seedance 可能将同一人生成两次、出现重复角色 | 每 phase 提示词中**明确画面人数**（如「仅两人、无其他人物」）；详见 [docs/19-画面人数明确.md](docs/19-画面人数明确.md) |

## 四、执行清单（新建或改写 section 时）

- [ ] **节奏规划**：在 README 或 00 中标出快段/慢段，并落实到各段台词时间块。
- [ ] **台词密度**：至少 1 段密集、1 段留白，各段句数/字数有差异。
- [ ] **情节说明**：每段含动作顺序 + 视线或微表情，不只一句概括。
- [ ] **提示词本段**：每段含可执行动作链与 A→B 互动，至少一处表情/神态变化描述。
- [ ] **寡言/内敛角色**：若存在此类人设，00 或 README 已澄清「寡言≠无反应」且**寡言≠只会说嗯**（亲密双人场合下须有有内容的短句）；秒级表该角色列每时间块有对对手方的具体微反应；【本段动作与节奏】显式写「对手做 X→该角色做 Y」的触发-反应链。
- [ ] **phase 衔接与过渡**：在 00 或 README 中为相邻 phase 选定或标注过渡方案（首尾帧 / 动作切点 / 景别/黑场/节奏等）；各 phase 边界写清段尾/段头动作或景别。
- [ ] **环环相扣**：每段情节说明中部分提及「承接上段末尾…」与「本段收束/下段将…」；提示词（若允许）可带一句前后衔接说明。
- [ ] **环境/氛围/打光统筹**：在 00 与 refs 中约定「第一段环境氛围参考图」；各段提示词中明确环境/光线/窗外天气与段 1 一致；段 2～12 上传该参考图以锁定氛围。
- [ ] **人与场景位置关系**：每段提示词中写出人物在场景中的具体位置（如窗左/沙发右/两人相对）；有运镜时显式约束运镜过程中人景相对位置不变；与 00 或第一段约定的空间布局一致；写下一段前通过**上一段尾帧**观察实际人景位置，据此调整本段提示词以保持空间延续。
- [ ] **门与环境设计**：涉及门进出时一律用**自动门（横向滑开/滑闭）**，不写推门/拉门/转门把；refs/scenes 或 00 中是否有**细致场景描述**（空间布局、门/窗/家具位置与样式、材质光线、动线），达到可据此画出一致画面的程度；见 [docs/16-环境与门的设计.md](docs/16-环境与门的设计.md)。
- [ ] **物品关系与情节连贯**：创作时考虑**位置关系、情节关系、物品关系**；关键物品/动作（行李、包、帽子等）在 section 内有简单时间线或标注；写每段前检查前面各 phase 是否已完成该动作或改变该物品状态，避免本段重复（如 phase02 已放好行李则 phase04 不再写「放好行李」）；见 [docs/15-物品关系与情节连贯.md](docs/15-物品关系与情节连贯.md)。
- [ ] **运镜/镜头/程度**：每段提示词写出景别与运镜类型+程度；00 或 README 可做本 section 运镜约定。
- [ ] **提示词块格式**：粘贴到 Seedance 的块按《phase-prompt-format》五段式；【本段动作与节奏】用时间块而非逐秒 15 行。
- [ ] **台词/对白**：有台词的段在粘贴块内包含「本段对白（须体现口型）」按时间块列出谁说什么，且【本段动作与节奏】中嵌入关键句台词（引号内）。
- [ ] **穿着状态**：若有明显穿着变化（如脱外套），脱之后各 phase 的提示词/情节说明中是否有男女主穿着状态的明确一句；ref 图是否与剧情阶段一致或已在提示词中强调。
- [ ] **全能参考**：是否拆成**两个独立视频**（7s 环境 + 8s 连续性）分别上传/使用，而非拼成一条 15s。
- [ ] **配乐**：若某段使用配乐，提示词中是否已写明「配乐在 15s 内封闭、结尾前有明确渐进终止，且最后 1 秒不能有音乐」；否则该段尽量不用配乐。
- [ ] **对话不跨 phase**：本 phase 开头是否出现对**上一 phase 最后一句台词**的直接回应（如「嗯」接上段）？若有，是否已把该回应挪到上段尾或把「上句+回应」放在同一 phase；见 [docs/17-对话回应不跨phase.md](docs/17-对话回应不跨phase.md)。
- [ ] **无模糊表述**：情节说明/物品/提示词中是否出现「A 或 B」「A/ B」类写法？若有，是否已改为唯一确定项并与 00、其他 phase 一致；见 [docs/18-禁止模糊表述.md](docs/18-禁止模糊表述.md)。
- [ ] **画面人数明确**：Seedance 提示词中是否已写明本段**画面内人数**（如「仅两人、无其他人物」），避免同一人重复出现；见 [docs/19-画面人数明确.md](docs/19-画面人数明确.md)。
