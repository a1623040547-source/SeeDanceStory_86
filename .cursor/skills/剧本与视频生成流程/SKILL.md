# 剧本与视频生成完整流程（项目内 Skill）

**说明**：**基于 API 生成连贯视频**（如 section04）已标记为废案，详见 `section04/废案说明.md`。本 Skill 中「新建 section、剧本、phase、refs、操作流程」等仍适用于在 **Seedance 平台手动逐段生成** 的 section（section01/02/03）。

本 Skill 总结从「构思剧本」到「可逐段复制到 Seedance 2.0 生成」的完整流程与规范，以 section01、section02 为参照。

---

## 一、何时使用本 Skill

- 用户要求**新建一个 section**（如 section03）并需要完整剧本 + 生成流程。
- 用户要求**为现有 section 补全**：refs 文件夹、本段情节说明、总述与规则写入每段提示词、操作流程文档。
- 用户询问**剧本/phase/总述/素材**应如何组织、命名或填写。

---

## 二、Section 目录结构规范

每个 section（如 `section02/`）应至少包含以下结构与文件：

```
sectionXX/
├── 00-项目总述与素材说明.md   # 总述全文 + 素材类型与衔接规则（角色图/场景图/尾帧图）
├── README.md                  # 项目概述 + 逐段剧本（画面/日文配音/时间轴）+ 剪辑总时间轴
├── 视频生成操作流程.md        # 从前期准备到导出成片的步骤
├── refs/                      # 启动前所需参照图（预留位置）
│   ├── characters/            # 角色形象图（定妆、86 或指定画风）
│   │   └── README.md          # 说明放置内容、命名、用法
│   └── scenes/               # 场景图 / 段 1 首图
│       └── README.md          # 说明放置内容、命名、用法
└── phases/                    # 每段生成用文件
    ├── phase01.txt
    ├── phase02.txt
    └── … (phase12.txt 等，按总时长与单段时长计算段数)
```

**要点**：
- **refs/characters/** 与 **refs/scenes/** 必须在项目启动前预留，并在 README 中说明「放什么、怎么用、命名建议」。
- 段数 = 总时长（秒）÷ 单段最大时长（如 15s）；段 1 为图生，段 2～N 为 V2V 接力。

---

## 三、00-项目总述与素材说明.md 必须包含

1. **一、生成用总述**  
   项目名、风格、角色概述（姓名/性格/声线/外观）、剧情概述（一句话或分步）、环境概述（场景/光线/景别）、氛围概述、配音配乐概述。  
   若 phase 内提示词需「每段必带总述」，此处为中文版原文；英文浓缩版写入各 phase 的 Seedance 提示词块开头。

2. **二、素材类型与衔接规则**  
   - 明确区分：**角色形象图**（refs/characters/）、**场景图/首图**（refs/scenes/）、**上一段尾帧图**（从上一段视频最后一帧截取）、**上一段完整视频**（可选）。  
   - 每段使用的素材一览表：本段用哪张图/哪段视频，本段生成后需产出哪张尾帧。  
   - 如何与下一段衔接：尾帧截取时机、提示词应写「接下来发生的动作」、保留 same scene / consistent character 等。  
   - **可选**：在创建剧本时即做「phase 衔接与镜头过渡总表」（每对相邻 phase 的过渡方式：首尾帧 / 动作切点 / 景别 / 黑场等），见《section-phase-script-retro》第五节。

3. **三、与本目录其他文件的关系**  
   简表：00、README、refs、phases、视频生成操作流程 各自作用。

---

## 四、README.md 必须包含

- 项目概述表：风格、场景、角色、剧情、单段时长、总时长、生成方式（段 1 图生，段 2～N V2V）。
- 生成核心参数：模型、时长、比例、V2V 强度、提示词原则。
- **逐段完整剧本**：每段含「画面」「日文配音·角色（语气）」「时间轴 + 日文 + 罗马音 + 中文」「Seedance 提示词」「负面词」（段 1 必写，段 2～N 可写「同段 1」）。
- **剪辑总时间轴**：序号、时间轴、内容简述，便于直接导入剪辑软件。

---

## 五、每个 phase 文件（phase01.txt～phaseNN.txt）必须包含

按顺序：

1. **标题行**  
   `# phaseXX | 段N | 图+文生视频 或 V2V接力 | 15s`

2. **【本段情节说明】**  
   一两句话概括本段在整剧中的作用：谁做了什么、情绪或转折。**为便于剧情与人物衔接自然，应部分提及**「承接上段末尾的什么状态」与「本段收束/下段将如何开场」，形成 phase 间环环相扣（详见《section-phase-script-retro》第六节）。必加，便于生成时理解上下文。

3. **【本段素材】**  
   本段使用的素材（首图 / 上一段尾帧图 文件名）；本段生成后需产出的尾帧文件名（若为最后一段则写「无需再截尾帧」）。

4. **【本段动作·演出·运镜·场景】**（推荐，二选一或混用）  
   可采用**分秒**或**分块**两种设计模式之一（见下方「phase 设计模式：分秒与分块」）。**此表不纳入给 Seedance 的整份提示词**；给 Seedance 的整份从【项目·风格·规则】到【负面词】连续书写，采用《phase-prompt-format》规定的结构，无分框说明。

5. **【本段剧本台词（配音/TTS 用）】**  
   时间轴（如 0～4s）、说话人及语气、日文、罗马音、中文。便于配音或 TTS 直接使用。

6. **给 Seedance 的整份提示词**（【项目·风格·规则】～【负面词】）  
   - 含 section 总述与规则，可引用 `@sectionXX/00-项目总述与素材说明.md`；**格式**见《phase-prompt-format》Skill。  
   - 内容为**整份可给 Seedance 的中文**（本项目采用中文以利过审与模型解析），结构为：  
     - **【项目·风格·规则】**：风格、角色、场景、总段数、一致性规则。  
     - **【场景与氛围】**：本段场景细节与氛围。  
     - **【景别与运镜】**：景别、运镜类型与程度。  
     - **【本段动作与节奏】**：按时间块（开场/中段/收尾）写主要动作与节奏，不用逐秒 15 行。  
     - **【台词概述】**：本段谁说了什么、顺序。  
     - **【负面词】**：需避免的画面或风格，逗号分隔。  
   **整份即 Seedance 所需全部内容，不写「复制到提示词框」「复制到负面词框」等分框说明。**

---

### phase 设计模式：分秒与分块

在同一段 15s 的 phase 内，对「本段动作·演出·运镜·场景」的细致划分可采用两种方式，**二选一或混用**（如前半按块、后半按秒）。生成时仍以**一整段 15s** 提交 Seedance，区别仅在于**写作与分镜的粒度**。

| 模式 | 划分依据 | 时间轴形式 | 适用场景 |
|------|----------|------------|----------|
| **分秒** | 按固定秒数 | 0–1s、1–2s … 14–15s，共 15 格 | 对每一秒都有明确动作/表情/运镜控制；分镜、对轨要求细到秒 |
| **分块** | 按**运动情节 / 镜头切换** | 若干**块**，每块有起止时间（如 0–3s、3–6s、6–10s、10–15s） | 按「动作转折、镜头切点、情绪切点」划分；块内可写细，块边界对应剪接/运镜变化 |

**分块模式要点**：

- **块仍有时间轴**：每块必须标注起止时间（如 块1：0–3s、块2：3–6s），保证整段 15s 被完整覆盖且不重叠或遗漏。
- **块的划分依据**：
  - **运动情节**：一个完整动作或动作阶段（如「入场→站定」「递出→对方接过」「从惊讶到憋笑」）；
  - **镜头切换情节**：景别变化、机位切点、运镜起止（如「中景固定→切特写→缓慢拉远」）。
- **在 phase 文件中的写法示例**（分块）：  
  `【本段动作·演出·运镜·场景（分块）】`  
  - 块1（0–3s）：角色推门入场，中景固定，视线扫过室内。  
  - 块2（3–6s）：切特写，表情从疑惑到恍然，微点头。  
  - 块3（6–10s）：拉至双人半身，A 递出物品，B 伸手接。  
  - 块4（10–15s）：固定机位，两人简短对话，收束在 B 转身。
- **与给 Seedance 的提示词**：无论分秒还是分块，粘贴到 Seedance 的【本段动作与节奏】仍用《phase-prompt-format》的**时间块**表述（开场/中段/收尾或按块概括），不把整张秒级/块级表贴入；块级设计可浓缩为「块1 入场—块2 特写反应—块3 双人递接—块4 对话收束」等一句或几句。

若某 section 采用分块模式，建议在 00 或 README 中注明「本 section 采用分块设计」，并在各 phase 内统一使用「【本段动作·演出·运镜·场景（分块）】」标题与块+时间轴格式。

## 六、视频生成操作流程.md 必须包含

- **〇、生成前必读**：总述与素材说明的位置、本段素材类型（避免尾帧图与角色图混用）。
- **一、前期准备**：剧本与时间轴、**角色形象图**（放入 refs/characters/）、**场景图/首图**（放入 refs/scenes/）、配音准备。
- **二、段 1**：图生视频，上传首图、时长、**从对应 phase 取给 Seedance 的整份提示词（含总述与负面词）**、截取尾帧。
- **三、段 2～N**：V2V，上传上一段尾帧图、**从对应 phase 取给 Seedance 的整份提示词**、截取尾帧。
- **四、导出与剪辑**：按时间轴拼接、对齐配音、导出成片。
- **五、常见问题**：崩脸、动作跳跃、时长与段数变更。
- **六、文件与命名建议**：00、refs/characters、refs/scenes、尾帧图、各段视频、phases、README、操作流程。

## 七、经验总结与注意事项

1. **角色与风格统一**：整 section 固定主角与画风（如 86 辛恩×蕾娜），总述中写清角色概述，每段提示词开头用同一套【项目·风格·规则】（见《phase-prompt-format》），可显著减少崩脸与风格漂移。

2. **每段必带总述与规则**：每段 Seedance 提示词**开头**都应带「【项目·风格·规则】」（见《phase-prompt-format》），再接场景、运镜、动作与节奏、台词概述，避免模型只看到本段而丢失前后一致性。采用**五段式结构化**便于 Seedance 2.0 学习。

3. **情节说明不可省**：每个 phase 的「本段情节说明」用于人工/AI 理解本段在剧情中的位置与情绪，便于改写提示词或检查衔接，不要只写画面不写情节。

4. **台词与时间轴必含**：每段 phase 内必须包含「本段剧本台词」：时间轴 + 谁 + 日文 + 罗马音 + 中文，便于配音与 TTS，也便于在提示词中写 Dialogue 概述。

5. **提示词「整份即用」**：用户应能打开对应 phase，将【项目·风格·规则】到【负面词】整份给 Seedance 即用，无需再打开 00 或自行拼接，也不区分提示词框/负面词框。因此总述与规则要写入 phase，而不是「请先复制 00 再复制本段」。

6. **refs 预留位置**：项目启动前就要有 `refs/characters/` 与 `refs/scenes/` 及各自 README，明确「角色形象图」「场景图/首图」的用途与命名，避免与「尾帧图」混淆。

7. **尾帧衔接规则**：上一段视频**最后一帧**作为下一段 V2V 的视觉起点（必须取最后一帧以保证连贯）；提示词描述「从上一帧之后接下来发生的动作」，并保留 same scene、consistent character、风格关键词。

8. **段数与总时长**：单段时长以平台为准（如 Seedance 2.0 当前 15s）；总时长 = 段数 × 单段时长；若改段数或总时长，需同步改 README、00、操作流程与各 phase 的段号与尾帧编号。

9. **phase 间镜头过渡（创建剧本时即规划）**：以 15s 为一 phase 时，除首尾帧 V2V 衔接外，在**创建剧本阶段**就为相邻 phase 设计多种可选的镜头过渡方案（如动作切点、景别变化、黑场/淡入淡出、情绪节奏切点、台词桥梁等），并在 00 或 README 中做「phase 衔接总表」、在各 phase 边界写清段尾/段头动作或景别。详见《section-phase-script-retro》Skill 第五节。

## 八、执行清单（新建 section 时）

- [ ] 创建 `sectionXX/` 及子目录 `refs/characters/`、`refs/scenes/`、`phases/`。
- [ ] 在 `refs/characters/` 与 `refs/scenes/` 下各写 README，说明放置内容与命名。
- [ ] 撰写 `00-项目总述与素材说明.md`（总述 + 素材类型与衔接规则 + 文件关系）。
- [ ] 撰写 `README.md`（概述 + 核心参数 + 逐段剧本 + 剪辑总时间轴）。
- [ ] 按段数创建 `phases/phase01.txt`～`phaseNN.txt`，每份含：本段情节说明、本段素材、（可选）本段**秒级或分块**演出表、本段剧本台词、**按《phase-prompt-format》五段式组织的 Seedance 中文提示词**、负面词。
- [ ] 撰写 `视频生成操作流程.md`，并确保其中引用 refs、00、phases 的用法与命名一致。
- [ ] 核对：每段提示词块是否按「项目·风格·规则 → 场景与氛围 → 景别与运镜 → 本段动作与节奏 → 台词概述」五段书写；是否注明见 `@sectionXX/00-项目总述与素材说明.md` 及《phase-prompt-format》。
- [ ] （可选）按《section-phase-script-retro》在 00 或 README 中做 phase 衔接总表，并为关键 phase 边界标注镜头过渡方案（首尾帧 / 动作切点 / 景别 / 黑场等）。

完成以上后，该 section 即具备「剧本完整 + 可直接复制到 Seedance 生成 + 角色与场景图预留」的完整流程。
