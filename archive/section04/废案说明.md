# Section04 废案说明

**状态：废案（2026-02）**

本 section 原计划通过 **Seedance 2.0 / 即梦 API** 生成连贯的 8 段视频（夏日海边·战后度假）。经实践，**使用 API 生成连贯视频在当前阶段尚不切实际**（提示词与参考图约束、单段时长与画质限制、连贯性与配音等），故将 section04 及与之配套的 API 视频流程相关 skills 标记为废案。

- **section04** 目录保留作参考（剧本、phase 提示词、已生成的首帧图与单段视频等），不再作为主推生成路径。
- 新建 section 或生成视频时，建议沿用 **section01 / section02 / section03** 的方式：在 Seedance 平台手动生成、逐段上传参考图与提示词，不依赖 API 自动化。

仅以下与「**用 API 生成连贯视频**」直接相关的 skill 标记为废案：

- **seedance-api**：通过 API 生成视频的约束与流程 → 废案

以下 skills **不依赖 API**，仍可用于在 Seedance 平台手动生成的 section（如 section01/02/03）：尾帧提取、phase 后期改名/合并、过渡补差等照常使用。

- **extract-video-tail**：段视频尾帧提取（V2V 用）→ 仍可用
- **phase-video-post**：phase 视频后期（改名/尾帧/合并）→ 仍可用
- **merge-video**：phase 视频合并测试 → 仍可用
- **phase-transition**：phase 间过渡补差 → 仍可用
- **剧本与视频生成流程**：新建 section 流程仍可用；其中「基于 API 生成连贯视频」方案废案。
